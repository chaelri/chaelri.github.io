01 · Overview

How a page turns

One button drives the entire UI. Each gesture (tap, double-tap, hold) maps to one move through the three-state machine: BOOK → CHAPTER → READING. Position is saved to NVS so it resumes on power-up.

arrow_forward Tap
Forward one step. In BOOK mode → next book. In CHAPTER mode → next chapter. In READING mode → next page.
arrow_back Double-tap
Backward one step. The firmware waits ~250 ms after a tap to see if a second tap arrives, then either commits the move or hops back two.
touch_double Hold
≥ 600 ms hold = select / advance to the next level. From READING, hold takes you back to the chapter picker. Hold ≥ 2 s anywhere = home (BOOK).
02 · Hardware

Top-down view

Three parts, six jumpers, plug in USB-C. The OLED runs on the 3.3 V rail; the button hangs off GPIO 4 with the ESP32-S3's internal pull-up. No external resistors, no level shifter, no enclosure required for v1.

developer_board
ESP32-S3 SuperMini
240 MHz dual-core, WiFi/BT (unused here), 4 MB flash (or 8 MB), ~22.5 × 18 mm. Verify your exact pinout — clone boards label the long edges slightly differently. 3V3, GND, IO 4, IO 8, IO 9 are the ones we need.
monitor
0.91" OLED · SSD1306
128 × 32 monochrome (yellow or white), I²C address 0x3C, 3.3–5 V input via onboard regulator. Pads usually labeled GND · VCC · SCK · SDA in that order — double-check your module since a minority swap GND/VCC.
smart_button
Tactile button (6×6×5 mm)
Any 2-circuit tactile pushbutton. INPUT_PULLUP on IO 4 means no external resistor; press shorts the pin to GND. ~30 ms software debounce; tap / double-tap / hold detected in readButton().
03 · Wiring

All six connections

Four jumpers to the OLED, two more to the button, then the USB-C cable. No external resistors, no level shifter, no transistor. The OLED runs from the ESP32's 3V3 rail and the button shares the same GND.

# Connection From To Wire Method
warning
Polarity matters
Most 0.91" modules use VCC = 3.3 V (regulated on-board). Swapping VCC ↔ GND can fry the SSD1306 driver. Verify the silkscreen on your specific module before powering up.
info
I²C pin freedom
GPIO 8 / 9 are the ESP32-S3's default I²C pins, but the Wire library accepts any GPIO via Wire.setPins(SDA, SCL). If your board exposes those pins awkwardly, swap to any other two GPIOs and update the sketch.

05 · Code

Firmware sketch

Skeleton only — full Arduino sketch (with U8g2 rendering, NVS-backed state, and the packed-NASB seek logic) will land in firmware/ereader.ino once we agree on the data-packing format.

// ereader.ino — sketch outline
#include <U8g2lib.h>
#include <LittleFS.h>
#include <Preferences.h>

const int SDA_PIN = 8;
const int SCL_PIN = 9;
const int BTN_PIN = 4;          // tactile button to GND

U8G2_SSD1306_128X32_UNIVISION_F_HW_I2C oled(U8G2_R0, U8X8_PIN_NONE);
Preferences prefs;

enum Mode { BOOK, CHAPTER, READING };
Mode mode = BOOK;
uint8_t bookIdx = 0;             // 0..65
uint8_t chapterIdx = 0;          // 0..N-1 of current book
uint16_t pageIdx = 0;            // page within current chapter

void setup() {
  pinMode(BTN_PIN, INPUT_PULLUP);
  Wire.setPins(SDA_PIN, SCL_PIN);
  oled.begin();
  LittleFS.begin();
  prefs.begin("ereader", false);
  bookIdx    = prefs.getUChar("book", 0);
  chapterIdx = prefs.getUChar("chap", 0);
  pageIdx    = prefs.getUShort("page", 0);
  render();
}

void loop() {
  Gesture g = readButton();      // NONE / TAP / DOUBLE / HOLD
  if (g != NONE) { handle(g); render(); savePos(); }
}
