# Flux — Digital Planner

**Task management with calendar, drag-drop scheduling, dark mode.** Last commit Apr 9, 2026 (parked).

**Stack:** Vanilla JS (~707 lines), Tailwind CSS, Material Icons, Workbox v7.0.0 PWA, Firebase RTDB.

## File Structure
```
flux/

├── script.js           (~707 lines) — Core logic, Firebase CRUD, drag system
├── style.css           (~448 lines) — Tailwind + custom, dark mode, drag styles

```

## Task Data Model (Firebase RTDB)
```js
{
  id: string,                   // Auto-generated by Firebase
  title: string,
  desc: string,
  date: string,                 // ISO YYYY-MM-DD (empty = unscheduled)
  time: string,                 // HH:MM (empty = all-day)
  allDay: boolean,
  color: string,                // blue|red|purple|green
  priority: string,             // high|medium|low|none
  isDone: boolean,
  subtasks: [{ text, done }]
}
```

## localStorage
- `theme`: `'dark'` | `'light'`
- `mainSort`, `tableSort` (sort prefs)

**No task data in localStorage** — all in Firebase.

## Firebase Realtime DB
- **Path:** `tasks/{id}` — flat collection
- **Listener:** `onValue(ref(db, "tasks"), cb)` — fires on any change
- **Sort:** Priority DESC, then time ASC

## Drag System (Pivotal)
1. **pointerdown:** captures position, sets `dragStartX/Y`
2. **pointermove:** if distance > 10px AND drag handle clicked, enter drag state (10px threshold prevents accidental drags from scroll)
3. **pointerup:** find drop zone (calendar cell, pending list, trash) → update task date or delete

**Drop targets:** `.cell-content`, `#pending-tasks-list`, `#mobile-pending-list`, `#trash-zone` (appears at bottom during drag, height 0→120px).

## Calendar Views
- **Month:** 7-column grid, 120px min height per cell
- **Week:** 7 columns flex, sticky day headers
- **Day:** Centered single-day view, max 4xl width
- **Mobile 3-row:** Mini calendar (#mobile-row-calendar) + task list (#mobile-row-details) + drawer (#mobile-row-dump, resizable)

## Hotkeys
- `m`: Month view, `w`: Week view, `d`: Day view, `t`: Today, `n`: Focus task input
- Disabled when input/textarea focused

## Why
- **Why Firebase RTDB (not localStorage):** Real-time sync across tabs/devices
- **Why no offline edit queue:** Simpler UI, most users have persistent connection
- **Why drag-drop scheduling:** Visual, intuitive, faster than modal forms
- **Why 3 calendar views:** Different planning needs (month overview, week deadlines, day focus)
- **Why mobile 3-row split:** No modal stacking, all sections visible
- **Why subtasks as array:** Simple toggle, progress bar
- **Why Tailwind + custom CSS:** Fast prototyping + complex layouts
- **Why no auth:** MVP scope, single user

## Limitations
