Overview

How a reading is taken

The ESP32-C3 reads two cheap signals every second and paints them on the 72×40 OLED. NTP gives us the clock; a 220k/100k resistor divider gives us the battery voltage; the TP4056's open-drain CHRG pad tells us whether USB-C is actually charging the cell.

1. Power flows
LiPo runs the device through the TP4056; USB-C tops up the cell while in use.
2. ESP samples
ADC1_CH3 reads the divider 16× per tick; GPIO4 watches CHRG.
3. OLED paints
Three-row layout: clock, date, battery icon + percent + state.
Hardware

Three boards + two resistors

Stacked top-to-bottom: ESP32-C3 + 0.42" OLED dev board, TP4056 USB-C charger / protection, and the LiPo cell. The only extra parts are the two 1/4 W resistors that form the battery-sense divider.

ESP32-C3 + 0.42" OLED
01space-style dev board. SSD1306 wired internally on GPIO5/6. GPIO3 + GPIO4 free for sense.
TP4056 + DW01
USB-C charge in, protected 3.0–4.2 V out. CHRG pad doubles as a charge-state signal.
220k + 100k resistors
1/4 W metal-film. Total 320k draw → ~13 µA continuous from the cell. Negligible.
Wiring

Six connections in total

Four wires for the power chain, plus two thin signal wires for the battery-sense tap and the CHRG line. The OLED is already wired on the dev board, so no I²C jumpers to run.

# From To Wire / part Notes
priority_high Before plugging the ESP32 in

The TP4056 output sits at whatever the LiPo is at (3.0–4.2 V). That is safe to put on the ESP32's 5V pin — the onboard LDO drops it to 3.3 V with ~250 mV of headroom. Do not connect it to the V3 pin; 4.2 V on the regulated 3.3 V rail exceeds the chip's spec.

Code

Firmware

One Arduino sketch in firmware/battery-monitor/battery-monitor.ino. Uses U8g2 for the OLED, analogReadMilliVolts for the eFuse-calibrated ADC, and configTime + NTP for the clock.

Battery read + percent mapping
16 samples averaged, divider compensated, clamped 0–100%
expand_more 
// Divider: V_pin = V_bat * 100k / (220k + 100k) = V_bat * 0.3125
// V_BAT_FULL/EMPTY are the endpoints we map 0..100% across.
const float DIVIDER_RATIO = 3.20f;
const float V_BAT_FULL   = 4.20f;
const float V_BAT_EMPTY  = 3.30f;

float readBatteryVolts() {
  uint32_t acc = 0;
  for (int i = 0; i < 16; i++) {
    acc += analogReadMilliVolts(BAT_ADC_PIN);
    delay(2);
  }
  float vPin = (acc / 16.0f) / 1000.0f;
  return vPin * DIVIDER_RATIO;
}

int voltsToPercent(float v) {
  float pct = (v - V_BAT_EMPTY) / (V_BAT_FULL - V_BAT_EMPTY) * 100.0f;
  if (pct < 0)   pct = 0;
  if (pct > 100) pct = 100;
  return (int)(pct + 0.5f);
}

Charging detection (TP4056 CHRG)
Open-drain pad + INPUT_PULLUP = one wire, no transistor
expand_more 
// INPUT_PULLUP + open-drain CHRG: LOW means a charge cycle is active.
// When the cell is full the TP4056 stops driving CHRG, so it floats HIGH
// and we read HIGH (= not charging, possibly "full" depending on V_bat).
pinMode(CHRG_PIN, INPUT_PULLUP);

bool isCharging() {
  return digitalRead(CHRG_PIN) == LOW;
}
