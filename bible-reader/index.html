01 · Overview

How a page turn happens

Press NEXT → ESP32 advances the page index → reads verses from SD (or HTTPS Range-fetches them from chaelri.github.io/devo/nasb2020.json) → repaints the e-ink panel.

touch_app Input
Two tactile buttons. Tap NEXT / PREV → one page. Hold 800 ms → jump one chapter. Wrapping is automatic: end of chapter falls into the next, end of book → next book.
swap_horiz Source
SD.begin(SD_CS) at boot. If the card mounts and /Bible/INDEX.txt exists → SD mode. Otherwise → WiFi stream. The day the SD card lands, no code changes.
tablet Render
GxEPD2 drives the SSD1680. Header bar shows Book Chapter; body wraps verses; footer shows page x/y + chapter z/Z. Full refresh per page (3–5 s) keeps ghosting at zero.
02 · Hardware

Top-down view

Four modules: the ESP32-S3 SuperMini in the center, the 2.13" e-paper panel above it, the SD module to the right, and two tactile buttons at the bottom. TP4056 + 1000 mAh LiPo power the whole thing off USB-C.

bolt
E-paper is 3.3 V only
VCC goes to the ESP32's 3V3 pad. Hooking it to 5V destroys the panel — that's the most expensive single mistake on this build.
share
E-paper + SD share the SPI bus
MOSI (IO11) and SCK (IO12) are wired to both modules. Each has its own CS — IO10 for the panel, IO6 for the card — so the bus is never contended. MISO (IO13) only matters for the SD card.

05 · Code

Firmware & tools

The Arduino sketch and the Node script that prepares the SD-card files. Two artifacts, one repo, zero build steps for the firmware.

firmware/bible-reader.ino
Arduino sketch · auto-detect SD or stream
expand_more

Required libraries: GxEPD2, ArduinoJson ≥ 7, plus the bundled SD, SPI, WiFi, HTTPClient, WiFiClientSecure, Preferences. Board: ESP32S3 Dev Module, PSRAM off.

download Download bible-reader.ino
tools/build-sd-card.mjs
Node script · convert NASB JSON → /Bible/*.txt
expand_more

Run node tools/build-sd-card.mjs from the repo root. Reads devo/nasb2020.json, emits sd-card/Bible/<Book>.txt (66 files) plus INDEX.txt. Drag the whole Bible/ folder onto the SD card root when it arrives.
